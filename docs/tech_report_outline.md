## 1. 研究背景与问题定义

### 1.1 教育领域的挑战
- 传统教学中，教师需要花费大量时间准备教学材料、总结知识点、出练习题
- K12和大学课程内容多样，涉及文字+图片+图表等多种模态信息
- 单一文本模态的AI工具无法充分利用教材中的视觉信息

### 1.2 多模态大模型的机遇
- 视觉-语言大模型（VL-LLM）的发展使AI能够同时理解文本和图像
- Qwen-VL等国产模型在中文场景下表现优异，成本可控
- 可以辅助教师自动生成教学资源，提升教学效率

### 1.3 项目目标
- 设计并实现一个支持多模态输入的跨模态教学辅助系统
- 功能覆盖：知识点总结、智能问答、练习题自动生成
- 面向K12和大学课程，支持PPT/PDF/图片等常见教学材料格式

## 2. 系统架构设计

### 2.1 整体架构
```
┌──────────────┐     ┌─────────────────┐     ┌──────────────┐
│  Gradio 前端  │ ──▶ │  Python 后端    │ ──▶ │  Qwen-VL API │
│  (Web界面)    │ ◀── │  (业务逻辑)      │ ◀── │  (DashScope)  │
└──────────────┘     └─────────────────┘     └──────────────┘
```

### 2.2 模块划分
- **输入处理模块** (`input_processor`): 解析PPT/PDF/图片，提取文本和图像
- **模型引擎模块** (`model_engine`): 封装Qwen-VL API调用，提供总结/问答/出题三个接口
- **工具模块** (`utils`): JSON读写、文件类型判断等公共函数
- **Web界面** (`app`): 基于Gradio的交互式前端

### 2.3 数据流设计
```
上传文件 → 解析提取(文本+图片) → 结构化JSON → Qwen-VL API → 解析输出JSON → 前端渲染
```

### 2.4 接口规范
- 描述三个核心数据接口格式（material / summary / quiz JSON schema）
- 见项目 README 中的接口规范部分

## 3. 多模态处理方法

### 3.1 文件格式处理
- **PDF**: 使用 pdfplumber 库提取每页文本
- **PPT**: 使用 python-pptx 库提取文本和内嵌图片
- **图片**: 使用 PIL 验证图片有效性，保留路径供Qwen-VL直接理解

### 3.2 多模态融合策略
- Qwen-VL 原生支持文本+图像联合输入
- 在 API 请求中同时传递提取的文字和原始图片
- 利用模型的视觉理解能力处理图表、公式等视觉信息

### 3.3 当前限制与改进方向
- PDF内嵌图片提取暂未实现（可用PyMuPDF补充）
- PPT幻灯片整页渲染需要外部工具（LibreOffice等）
- 当前提供手动上传图片的备选路径

## 4. 模型调用与Prompt设计

### 4.1 模型选型
- 选择 qwen-vl-plus（阿里云DashScope）
- 理由：中文能力强、支持多模态、价格合理、API稳定
- 对比：vs GPT-4V (贵)、vs 开源部署 (硬件门槛高)

### 4.2 Prompt工程
- **总结Prompt**: 分步骤指令 + JSON输出格式约束 + 长度限制
- **问答Prompt**: 基于材料的回答约束 + 诚实性要求 + 语言适配
- **出题Prompt**: 题目数量/格式/难度的结构化指令 + JSON Schema约束
- JSON格式通过代码层正则解析做容错，兼容模型偶尔输出的markdown包裹

### 4.3 输出解析与容错
- 正则提取json代码块
- 降级处理（JSON解析失败时返回原始文本）
- Mock数据兜底（API不可用时的Demo模式）

## 7. 总结与展望

### 7.1 项目总结
- 成功实现了基于Qwen-VL的跨模态教学辅助系统
- 三个核心功能（总结/问答/出题）均已完成
- 系统界面友好，支持多种教学材料格式
- 通过Prompt工程实现了稳定结构化的输出

### 7.2 遇到的挑战
- PPT整页渲染的跨平台兼容性问题
- 模型JSON输出的稳定性控制
- 长文本材料的上下文截断处理

### 7.3 未来改进方向
- 支持更多模态（音频讲解、视频课程等）
- 增加RAG（检索增强生成）能力，支持多文档综合
- 接入本地模型（降低成本，保护隐私）
- 增加学生端功能（学习进度追踪、错题本等）
- 支持更多题型（判断题、填空题、主观题等）
- 优化长文本分块策略，支持整本教材的处理
